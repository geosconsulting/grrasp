The data stored in the local database and used by the platform are organized in:


  1. Workspaces
  2. Datastore
  3. Layers

Workspaces are group of data related with a particular project (e.g. gisplatform) while datastores are logical groups of data within a certain project (e.g. gas network data, electrical data, administrative data,etcâ€¦). The layers (e.g. airports, railroads,etc..) are the actual data each of which belongs to a workspace which in turn belongs to a datastore Workspaces can be added or deleted, datastores can be deleted and both of them can be refreshed when any changes is performed. In uploading external GIS files to the platform (zipped shapefile format) the data will be renamed by the server by adding a sequence with random letters and numbers using this code to create a new datastore. This uploading procedure will automatically create a datastore that will belong to the workspace active at the moment of uploading while the name of the layer will be the name of the file contained in the compressed file.




Selecting the new automatically created datastore the data contained in it will be made available in the layers section with the possibility to add it to the map.
